Control plane and scalability testing with kube-burner

Overview

Modern Kubernetes and OpenShift environments operate at massive scale, where even small inefficiencies in the control plane can lead to significant reliability, performance, and cost issues. Continuous control plane and scalability testing is essential to ensure that clusters remain stable and responsive as workload density, node count, and API activity increase.

This tutorial introduces kube-burner, a CNCF project with advanced performance and scale testing toolset designed to generate realistic k8s workloads and stress patterns. You will learn how kube-burner can be used to validate cluster scalability, uncover regressions, measure key performance indicators (KPIs), and provide actionable insights for both infrastructure and platform teams.

What you’ll learn

By the end of this tutorial, you will be able to:

  • Understand the role of control plane and scalability testing in Kubernetes and OpenShift.

  • Set up and configure kube-burner for large-scale workload generation.

  • Execute benchmark and custom workload scenarios to simulate real-world cluster behavior.

  • Measure and analyze some key performance metrics such as:

    • API server latency and throughput

    • Etcd performance

    • Runtime measurements like pod readiness latency

    • Control plane resource utilization

  • Ways to collect and visualize kube-burner metrics.

  • Integrate kube-burner into CI/CD pipelines for Continuous Performance Testing (CPT).

  • Interpret results to drive capacity planning, tuning, and product readiness decisions.

Conclusion

Scalability and control plane performance are foundational to the reliability of any Kubernetes-based platform. kube-burner enables teams to move from reactive performance testing to a proactive, automated, and data-driven testing strategy. By combining large-scale workload generation, metrics collection, and deep observability, kube-burner helps ensure that your platform remains performant as features, releases, and customer usage evolve.

Adopting kube-burner as part of your continuous testing strategy strengthens product quality, reduces production risk, and accelerates confident release cycles.

Next steps

  • Start with the kube-burner installation and setup guide.

  • Learn to configure and simulate a churn job.

  • Connect test results to monitoring dashboards (Prometheus, Grafana, Elasticsearch, etc.) for deeper analysis.

  • Try kube-burner hands on in the lab environment.
